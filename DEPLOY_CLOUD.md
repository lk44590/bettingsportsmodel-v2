# Cloud Deployment Guide

This guide will help you deploy the betting model to run automatically in the cloud without your computer needing to be on.

## Option 1: GitHub Pages + GitHub Actions (Recommended - Free)

### Step 1: Create GitHub Repository
1. Go to https://github.com/new
2. Create a new repository (name it something like "betting-model")
3. Don't initialize with README (we'll upload files)

### Step 2: Upload Files
1. Go to the repository you just created
2. Click "uploading an existing file"
3. Upload ALL files from the "Betting model upgraded" folder EXCEPT:
   - output/ folder (will be generated by the workflow)
   - tracking/ folder (will be generated)
   - __pycache__/ folder
   - .venv/ folder
4. Commit the files

### Step 3: Enable GitHub Pages
1. Go to repository → Settings → Pages
2. Under "Source", select "GitHub Actions" (or "Deploy from a branch" and select "main" folder "/output")
3. Save

### Step 4: Configure Secrets (for Odds API)
1. Go to repository → Settings → Secrets and variables → Actions
2. Click "New repository secret"
3. Name: `THE_ODDS_API_KEY`
4. Value: Your API key from provider_credentials.json
5. Click "Add secret"

### Step 5: Enable GitHub Actions
1. Go to repository → Actions tab
2. You should see the "Generate Daily Betting Card" workflow
3. Click "Enable workflow" if needed

### Step 6: Test Manual Run
1. Go to Actions tab
2. Click "Generate Daily Betting Card"
3. Click "Run workflow" → "Run workflow"
4. Wait for it to complete (2-3 minutes)

### Step 7: Access Your Site
Your site will be available at: `https://yourusername.github.io/betting-model/`

The daily card will be at: `https://yourusername.github.io/betting-model/output/daily-card-YYYY-MM-DD.html`

### Step 8: Add to Phone
1. Open the URL on your phone
2. Tap Share → "Add to Home Screen"
3. Works like a native app!

---

## How It Works

- **GitHub Actions** runs the generate_daily_card.py script daily at 10:00 AM Eastern
- The script generates HTML/JSON files in the output/ folder
- The workflow commits these files back to the repository
- GitHub Pages serves the HTML files to the web
- You can access from any device, anywhere

## Troubleshooting

**Workflow failing?**
- Check Actions tab for error logs
- Make sure THE_ODDS_API_KEY secret is set correctly
- Verify all files were uploaded correctly

**Site not loading?**
- Check GitHub Pages settings
- Make sure the workflow completed successfully
- Check the file path in the URL

**Want to update manually?**
- Go to Actions tab → "Run workflow" → "Run workflow"
- New card will generate in 2-3 minutes
